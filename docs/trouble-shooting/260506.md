# 트러블슈팅 — 2026-05-06

## 1. tsconfig `@figma/plugin-typings` 참조 오류

### 증상

`pnpm typecheck` 실행 시 다음 에러 발생:

```
error TS2688: Cannot find type definition file for '@figma/plugin-typings'.
```

### 원인

`tsconfig.json`의 `types` 배열은 npm 패키지 전체 이름이 아닌, **각 `typeRoots` 디렉토리 안에서의 폴더명**으로 해석된다.

```json
"typeRoots": ["./node_modules/@types", "./node_modules/@figma"],
"types": ["@figma/plugin-typings"]  // 잘못된 값
```

TypeScript는 `@figma/plugin-typings`를 다음 경로에서 탐색:
- `./node_modules/@types/@figma/plugin-typings` → 없음
- `./node_modules/@figma/@figma/plugin-typings` → 없음 (중복 스코프)

### 해결

`types` 값에서 `@figma/` 스코프를 제거한다.

```json
"types": ["plugin-typings"]  // 올바른 값
```

TypeScript가 `./node_modules/@figma/plugin-typings`를 정상 탐색하여 해결.

## 2. `/commit` 스킬 — mixed types 분리 커밋 시 의도치 않은 파일 포함

### 증상

mixed types 분리 커밋 진행 중 `git add tsconfig.json`만 실행했음에도 나머지 파일 6개가 함께 포함되어 단일 커밋으로 묶임.

```
fix(config): correction of @figma/plugin-typings reference in tsconfig
 7 files changed  ← 의도한 건 1개였음
```

### 원인

VSCode 등 IDE가 편집된 파일을 자동으로 staging area에 올리거나, 이전 작업의 staged 상태가 남아 있는 경우 `git add <file>` 이후 검증 없이 커밋하면 예상 외 파일이 함께 포함됨.

### 해결

`/commit` 스킬 Step 5 (mixed types 처리)에 두 단계 추가:

1. 각 그룹 커밋 전 `git reset HEAD -- .`으로 staging area 초기화
2. `git add <files>` 후 `git diff --staged --name-only`로 실제 staged 파일 검증
   - 예상 외 파일이 포함되어 있으면 커밋 중단 후 사용자에게 경고

### 복구 절차

```bash
git reset HEAD~1       # 커밋 취소 (변경사항 보존)
# 이후 그룹별로 reset → add → verify → commit 순서로 재진행
```
