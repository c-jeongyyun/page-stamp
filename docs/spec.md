# Figma 페이지 번호 플러그인 기획서

> 작성일: 2026-03-17

---

## 1. 개요

### 1.1 배경

Figma Design 파일로 포트폴리오 및 슬라이드 자료를 제작하는 디자이너들은 슬라이드에 페이지 번호를 붙여야 하는 상황을 자주 마주한다. 그러나 기존에 존재하는 페이지 번호 플러그인들은 아래와 같은 문제를 공통적으로 안고 있다.

- 프레임 이름을 특정 규칙에 맞게 변경해야 하는 번거로움
- 추가되는 페이지 번호 요소를 스타일링할 수 없음
- 페이지 번호 요소의 포지셔닝 방식을 제어할 수 없음

### 1.2 Figma Slides가 대안이 되는가?

Figma Slides는 네이티브로 슬라이드 번호 기능을 제공한다. 그러나 다음과 같은 이유로 Figma Design 사용자들에게 완전한 대안이 되지 못한다.

- **변환 조건 제한**: Figma Design → Figma Slides 변환은 4:3 ~ 16:9 종횡비의 최상위 프레임에서만 가능하며, 비표준 크기의 포트폴리오는 변환 자체가 불가하다.
- **기능 손실**: 변환 시 프레임 간 프로토타입 연결이 소실된다.
- **작업 환경 단절**: Figma Slides는 캔버스 크기가 1920×1080으로 고정되어 있고, 오토레이아웃·컴포넌트 등 Figma Design의 고급 기능 일부를 제한적으로만 사용할 수 있다.
- **번거로운 전환 비용**: 이미 Figma Design에서 작업 중인 파일을 오직 페이지 번호 하나를 위해 Slides로 변환하는 것은 비효율적이다.

따라서 **Figma Design 환경 안에서 페이지 번호를 자동화하는 플러그인은 여전히 유효한 수요**를 갖는다.

## 3. 문제 정의 및 해결 방향

### 문제 1. 프레임 넘버링 강제

**현상**
기존 플러그인은 `1#`, `2#` 식으로 프레임 이름을 변경해야만 해당 순서를 인식하고 페이지 번호를 붙여준다.

**문제점**
- 이미 의미 있는 이름이 붙은 프레임을 일괄 수정해야 한다.
- 작업 흐름이 끊기고, 프레임 이름이 오염된다.

**해결 방향**
프레임 이름에 의존하지 않고, **캔버스 상의 TL(Top-Left) 방향 위치 기준으로 프레임 순서를 자동 감지**하여 페이지 번호를 할당한다.

---

### 문제 2. 페이지 번호 스타일링 불가

**현상**
기존 플러그인이 생성하는 페이지 번호 요소는 단순 텍스트 레이어로, 각 슬라이드에 개별 삽입된다. 스타일을 바꾸려면 모든 슬라이드를 하나씩 수동으로 수정해야 한다.

**문제점**
- 폰트, 색상, 크기 등 스타일 일괄 변경이 불가능하다.
- 디자인 시스템과 분리된 요소가 되어 유지보수가 어렵다.

**해결 방향**
사용자가 사전에 **페이지 번호 마스터 컴포넌트**를 만들어두면, 플러그인은 해당 컴포넌트의 **인스턴스**를 각 슬라이드에 삽입한다. 이후 마스터 컴포넌트를 수정하면 모든 슬라이드의 페이지 번호에 스타일이 일괄 반영된다.

---

### 문제 3. 포지셔닝 방식 제어 불가

**현상**
일부 플러그인은 x, y 값을 입력해도 프레임의 오토레이아웃 설정 때문에 페이지 번호 요소가 의도하지 않은 위치에 삽입된다.

**문제점**
- 오토레이아웃이 적용된 프레임에서는 삽입된 요소가 레이아웃 흐름에 따라 배치되어 위치가 틀어진다.
- `absolute` 포지셔닝이 강제되지 않으면 좌표 입력이 무의미해진다.

**해결 방향**
플러그인 UI에서 **삽입 방식 선택 옵션**을 제공한다.
- **Absolute 모드**: 입력한 x, y 좌표에 절대 위치로 삽입 (`layoutPositioning: 'ABSOLUTE'` 적용)
- **Auto Layout 모드**: 프레임의 레이아웃 흐름에 따라 일반 요소로 삽입

## 4. 핵심 기능 명세

### 4.1 프레임 순서 자동 감지 (TL 정렬)

- 선택된 프레임들의 `x`, `y` 좌표를 읽어 **위→아래, 왼쪽→오른쪽** 순으로 정렬
- 정렬 결과를 플러그인 UI에 미리보기로 표시하여 사용자가 순서를 확인할 수 있도록 함
- 정렬 기준 커스터마이징 옵션 제공 (예: y 우선 정렬 / x 우선 정렬)

### 4.2 컴포넌트 인스턴스 삽입

- 플러그인 실행 시, 현재 파일 내 컴포넌트 목록을 조회하여 사용자가 페이지 번호용 컴포넌트를 선택할 수 있도록 드롭다운 제공
- 선택한 컴포넌트의 인스턴스를 각 슬라이드에 삽입
- 컴포넌트 내부의 특정 텍스트 레이어에 페이지 번호 값을 자동으로 주입
  - 텍스트 레이어 이름 규칙: `{page_number}` (기본값, 커스터마이징 가능)

#### 기본 컴포넌트 자동 생성

파일 내에 사용 가능한 컴포넌트가 없거나, 별도로 만들고 싶지 않은 경우 플러그인이 기본 컴포넌트를 자동으로 생성해주는 기능을 제공한다.

**컴포넌트 선택 방식 (라디오 버튼)**

STEP 2는 두 가지 선택지를 라디오 버튼으로 제공하며, 기본값은 "파일 내 컴포넌트 사용"이다.

- **파일 내 컴포넌트 사용 (기본 선택)**: 현재 파일 내 컴포넌트 목록을 드롭다운으로 제공. 텍스트 레이어명 입력 필드 함께 표시
- **기본 컴포넌트 자동 생성**: 선택 시 드롭다운 대신 하단에 인라인 안내 문구를 표시

**인라인 안내 문구 (기본 컴포넌트 자동 생성 선택 시)**

별도 모달 없이, 라디오 버튼 하단에 바로 노출한다.

- 컴포넌트가 **TL 정렬 기준 첫 번째 프레임의 오른쪽**에 생성된다는 위치 안내
- 생성 후 컴포넌트를 직접 수정하면 모든 페이지 번호에 스타일이 일괄 반영된다는 사용법 안내

**배치 위치**

TL 정렬 기준 첫 번째 프레임의 우측에 일정 간격(40px)을 두고 배치한다. 사용자가 캔버스에서 즉시 발견하고 스타일을 수정할 수 있도록 의도적으로 작업 영역 근처에 노출한다.

**기본 컴포넌트 스펙**

| 속성 | 값 |
|------|------|
| 컴포넌트명 | `PageNumber / Default` |
| 크기 | 60 × 32px |
| 배경 | 검정 (#000000), 모서리 반경 4px |
| 텍스트 레이어명 | `{page_number}` |
| 폰트 | Inter Regular 14px, 흰색 |
| 레이아웃 | 오토레이아웃 (가로/세로 중앙 정렬, 패딩 12/8px) |

### 4.3 포지셔닝 방식 선택

| 옵션 | 동작 |
|------|------|
| Absolute | `layoutPositioning: 'ABSOLUTE'` 적용 후 x, y 좌표 고정 |
| Auto Layout | 일반 자식 요소로 삽입, 레이아웃 흐름 유지 |

### 4.4 좌표 입력

- x, y 값을 직접 입력 (픽셀 단위)
- 자주 쓰는 위치 프리셋 제공: 좌하단, 우하단, 좌상단, 우상단, 중앙하단

### 4.5 페이지 번호 포맷

- 단순 번호: `1`, `2`, `3` …
- 전체 포함: `1 / 10`, `1 of 10` …
- 시작 번호 설정 (예: 표지 제외 시 2번 슬라이드를 `1`로 시작)
- 특정 슬라이드 제외 옵션

### 4.6 재실행 및 업데이트

**적용하기 vs 다시 적용(갱신) 버튼 분리**

| 버튼 | 동작 |
|------|------|
| 적용하기 | 선택된 프레임 전체에 페이지 번호 인스턴스를 새로 삽입 |
| 다시 적용 | 기존에 삽입된 인스턴스의 번호 텍스트만 갱신 (재삽입 없음) |

**갱신 시 번호 결정 원칙**

갱신은 "인스턴스가 존재하는 프레임"이 아닌, **"인스턴스가 있든 없든 전체 프레임의 TL 위치"** 를 기준으로 번호를 결정한다.

예시: 프레임이 왼→오로 1, 2, 3 순서로 배치되어 있고, 2번 프레임의 인스턴스를 수동으로 삭제한 경우
- 1번 프레임 인스턴스 → **1**
- 2번 프레임 인스턴스 없음 → **건너뜀** (재삽입 안 함)
- 3번 프레임 인스턴스 → **3**

즉, 번호는 프레임의 위치 순서가 결정하고, 인스턴스의 유무는 번호 값에 영향을 주지 않는다.

**인스턴스 식별 방식**

플러그인이 삽입한 인스턴스에는 `setPluginData`로 태그를 심어 재실행 시 정확히 식별한다. 사용자가 레이어 이름을 바꿔도 탐지가 깨지지 않는다.

```typescript
// 삽입 시
instance.setPluginData('isPageNumber', 'true');

// 갱신 시 — 태그된 인스턴스만 탐색
const existing = frame.findAll(
  (node) => node.getPluginData('isPageNumber') === 'true'
);
```

**전체 제거**

`setPluginData`로 태깅된 인스턴스를 모든 프레임에서 일괄 탐색 후 삭제한다.

## 6. 기술 구현 포인트

### 6.1 TL 방향 정렬 로직

```typescript
// 프레임을 y 우선, x 차순으로 정렬
const sortedFrames = frames.sort((a, b) => {
  if (Math.abs(a.y - b.y) < THRESHOLD) return a.x - b.x; // 같은 행
  return a.y - b.y;
});
```

### 6.2 기본 컴포넌트 생성 및 배치

```typescript
async function createDefaultPageNumberComponent(frames: FrameNode[]) {
  // TL 정렬 기준 첫 번째 프레임 기준으로 배치
  const firstFrame = frames[0];

  const component = figma.createComponent();
  component.name = "PageNumber / Default";
  component.resize(60, 32);

  // 배경 Rectangle
  const bg = figma.createRectangle();
  bg.resize(60, 32);
  bg.cornerRadius = 4;
  bg.fills = [{ type: 'SOLID', color: { r: 0, g: 0, b: 0 } }];
  component.appendChild(bg);

  // 텍스트 레이어 (번호 주입 타겟)
  const text = figma.createText();
  await figma.loadFontAsync({ family: "Inter", style: "Regular" });
  text.name = "{page_number}";
  text.characters = "1";
  text.fills = [{ type: 'SOLID', color: { r: 1, g: 1, b: 1 } }];
  text.fontSize = 14;
  component.appendChild(text);

  // 오토레이아웃 중앙 정렬
  component.layoutMode = 'HORIZONTAL';
  component.primaryAxisAlignItems = 'CENTER';
  component.counterAxisAlignItems = 'CENTER';
  component.paddingLeft = component.paddingRight = 12;
  component.paddingTop = component.paddingBottom = 8;

  // 첫 번째 프레임 오른쪽에 40px 간격으로 배치
  const GAP = 40;
  component.x = firstFrame.x + firstFrame.width + GAP;
  component.y = firstFrame.y;

  return component;
}
```

### 6.3 컴포넌트 인스턴스 삽입 + Absolute 포지셔닝

```typescript
const instance = component.createInstance();

if (positioningMode === 'ABSOLUTE') {

```

### 6.4 텍스트 레이어에 번호 주입

```typescript
const textLayer = instance.findOne(

await figma.loadFontAsync(textLayer.fontName as FontName);
```

### 6.5 갱신(다시 적용) 로직

번호는 인스턴스 유무와 무관하게 전체 프레임의 TL 순서 인덱스로 결정된다.

```typescript
async function refreshPageNumbers(frames: FrameNode[]) {
  // 1. 전체 프레임을 TL 순으로 정렬 → 인덱스가 곧 페이지 번호
  const sortedFrames = sortByTL(frames);

  for (let i = 0; i < sortedFrames.length; i++) {
    const frame = sortedFrames[i];
    const pageNumber = i + startNumber; // TL 위치 기반 번호

    // 2. 이 프레임에 플러그인이 심은 인스턴스가 있는지 확인
    const existing = frame.findOne(
      (node) => node.getPluginData('isPageNumber') === 'true'
    );

    if (!existing) continue; // 인스턴스 없으면 건너뜀 (재삽입 안 함)

    // 3. 있으면 번호 텍스트만 갱신
    const textLayer = existing.findOne(
      (node) => node.type === 'TEXT' && node.name === '{page_number}'
    ) as TextNode;

    await figma.loadFontAsync(textLayer.fontName as FontName);
    textLayer.characters = String(pageNumber);
  }
}
```

## 8. 플러그인 유효성 근거 요약

1. **Figma Slides는 완전한 대안이 아님**: 비표준 비율 프레임은 변환 불가, 프로토타입 연결 소실, Slides 고유의 기능 제약 존재
2. **기존 플러그인들의 한계**: 프레임 이름 규칙 강제, 스타일링 불가, 포지셔닝 제어 없음이라는 세 가지 공백이 해결되지 않음
3. **커뮤니티 수요 검증**: Figma 공식 포럼에서 자동 페이지 번호 기능에 대한 요청이 수백 건 이상 존재하며, 이는 Figma Design 사용자들의 실질적인 수요를 방증함
4. **포트폴리오로서의 가치**: 실제 문제를 해결하는 실용적인 결과물로, 프론트엔드/디자인 엔지니어링 직군의 취업 포트폴리오로 활용 가능
